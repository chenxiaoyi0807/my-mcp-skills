这项技能指导创建独特的、生产级的前端界面，避免通用的“AI slop”美学。实现真正的工作代码，特别注意美学细节和创意选择。

用户提供前端需求：要构建的组件、页面、应用程序或界面。它们可能包括有关目的、受众或技术限制的上下文。

## 设计思维

在编码之前，了解上下文并致力于大胆的美学方向：
- **目的**：这个接口解决什么问题？谁使用它？
- **色调**：选择一个极端：简约、忙碌、复古未来、有机/自然、奢华/精致、俏皮/玩具、社论/杂志、野兽派/原始、装饰艺术/几何、粉彩/粉彩、工业/实用等。有多种口味可供选择。使用它们获取灵感，但以美学方向设计产品。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异**：是什么让这令人难忘？人们会记住的一件事是什么？

**关键**：选择明确的概念方向并精确执行。大胆的极简主义和复杂的极简主义都有效——关键是意图，而不是强度。

然后实现工作代码（HTML/CSS/JS、React、Vue 等），即：
- 生产级和功能性
- 具有视觉冲击力且令人难忘
- 有清晰的审美观点
- 每个细节都经过精心打磨

## 前端审美指南

重点关注：
- **版式**：选择美观、独特且有趣的字体。避免使用 Arial 和 Inter 等通用字体；相反，选择独特的选择来增强前端的美感；意想不到的、独特的字体选择。将独特的显示字体与精致的正文字体配对。
- **颜色和主题**：旨在实现有凝聚力的美感。使用 CSS 变量来保持一致性。带有大胆强调的原色击败了胆怯、间隔均匀的调色板。
- **运动**：使用动画来实现效果和微交互。与 HTML 相比，更喜欢纯 CSS 解决方案。如果可用，请使用 React 运动库。关注高影响力的时刻：精心策划的页面加载和交错显示（动画延迟）比分散的微交互更有趣。使用令人惊讶的滚动触发器和悬停状态。
- **空间构图**：意外的布局。不对称。重叠。对角流。打破网格的元素。慷慨的负空间或受控的密度。
- **背景和视觉细节**：创建氛围和深度，而不是默认为纯色。添加符合整体美感的上下文效果和纹理。应用创意形式，例如渐变网格、噪声纹理、几何图案、分层透明度、戏剧性阴影、装饰性边框、自定义光标和粒子叠加。

切勿使用人工智能生成的通用美学，例如过度使用的字体系列（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（尤其是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏上下文特定特征的千篇一律的设计。

创造性的诠释和意想不到的选择让人感觉真正是为环境而设计的。任何设计都不应该是相同的。浅色和深色主题之间存在差异，字体不同，审美也不同。永远不要在跨代的共同选择上趋同（比如 Space Grotesk）。

**重要**：将实现复杂性与审美愿景相匹配。极繁主义设计需要包含大量动画和效果的复杂代码。极简主义或复杂的设计需要克制、精确，并仔细注意间距、排版和微妙的细节。优雅源自良好的视野。
